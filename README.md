# Свадебное приглашение — Алексей и Екатерина

Статический сайт-приглашение: 4 стилевых варианта, печать в чистый A4-PDF,
визуальный редактор текста и хранение правок в базе (Vercel KV) — заказчик
меняет текст без правки кода и без гита, гости видят актуальную версию.

## Структура

```
index.html            галерея — выбор варианта + кнопка «Редактировать текст»
editor.html           визуальный редактор: поля + живой превью + сохранение
content.js            текст по умолчанию (fallback) — общий для всех вариантов
render.js             подстановка текста из БД в вариант + живой превью
variants/             4 варианта (печать → 2 страницы A4)
  01-ink.html         Ink — Cormorant Garamond + Lora (айвори/чернила)
  02-gold.html        Gold — Forum + PT Serif (золото, двойная рамка)
  03-sage.html        Sage — Playfair Display + Lora (зелёный)
  04-terracotta.html  Terracotta — Prata + PT Serif (терракота)
api/content.js        serverless: GET читает текст из KV, POST сохраняет (пароль)
vercel.json           cleanUrls
```

## Как это работает

- Гость открывает ссылку варианта → `render.js` тянет текст из `/api/content`
  (Vercel KV). Нет данных/интернета → показывается текст из `content.js`.
- Заказчик открывает `editor.html`, правит поля (справа живой превью),
  жмёт «Сохранить» (вводит пароль) → текст уходит в KV. Гости сразу видят новый.
- «Скачать PDF» — печать текущего варианта (Сохранить как PDF, A4, поля «Нет»).
- «Ссылка гостям» — копирует адрес выбранного варианта для рассылки.

## Деплой на Vercel

1. **Импорт проекта.** Vercel → Add New → Project → импортировать репозиторий
   `Acron1k/wedding_gess_site`. Framework Preset: **Other**. Deploy.

2. **Подключить базу (Vercel KV / Upstash).**
   В проекте → вкладка **Storage** → **Create Database** → **KV (Upstash Redis)** →
   создать и **Connect** к проекту. Vercel сам добавит переменные окружения
   `KV_REST_API_URL` и `KV_REST_API_TOKEN`.

3. **Задать пароль редактора.**
   Project → **Settings → Environment Variables** → добавить
   `EDIT_SECRET` = ваш_пароль (его вводят в редакторе при сохранении).

4. **Redeploy** (Deployments → ⋯ → Redeploy), чтобы переменные применились.

После деплоя:
- галерея: `https://<проект>.vercel.app/`
- редактор: `https://<проект>.vercel.app/editor`
- ссылка гостям (пример): `https://<проект>.vercel.app/variants/01-ink.html`

## Локальный просмотр

```
python -m http.server 4321
```
Откройте `http://localhost:4321/`. Локально база недоступна — показывается
текст по умолчанию из `content.js`, живой превью в редакторе работает.

## Менять текст по умолчанию

Правьте `content.js` (значения в кавычках). Это нужно редко — обычно текст
меняется через `editor.html`.
